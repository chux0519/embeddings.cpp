# Alignment and Benchmark Tool

`scripts/alignment.py` compares embeddings generated by:

- Python `transformers` on CPU
- `embeddings.cpp` GGUF inference
- Hugging Face `text-embeddings-inference` (TEI), when enabled

The default model list is parsed from `README.md`, so the script stays aligned with the models documented by this repo.

## Quick Start

Build and install the local extension first:

```bash
cmake -S . -B build -DEMBEDDINGS_CPP_ENABLE_PYBIND=ON -DEMBEDDINGS_CPP_BUILD_WASM_TOOLS=OFF -DGGML_NATIVE=OFF -DGGML_OPENMP=OFF
cmake --build build -j
uv pip install --torch-backend cpu -r scripts/requirements.txt
uv pip install -e .
```

Run correctness checks for all README models that already have `models/*.fp16.gguf` files:

```bash
uv run scripts/alignment.py
```

Convert missing fp16 GGUF files before testing:

```bash
uv run scripts/alignment.py --convert-missing
```

Run correctness plus CPU performance benchmarks:

```bash
uv run scripts/alignment.py --benchmark --iterations 10 --warmup 2
```

Run the focused BGE-M3 evaluation with isolated Python CPU and
`embeddings.cpp` worker processes:

```bash
uv run scripts/bge_m3_eval.py \
  --convert-missing \
  --batch-sizes 1 4 8 \
  --iterations 5 \
  --warmup 1
```

This writes `scripts/output/bge_m3_eval_*.json` and `.md` reports with:

- Python CPU vs `embeddings.cpp` cosine correctness
- `embeddings.cpp` batch-vs-single cosine correctness
- single-request (`batch=1`) and batch latency/throughput
- isolated load RSS and benchmark peak RSS for each runner
- a conclusion that says whether to optimize batch throughput first or start
  quantization next

Run Snowflake with TEI as an additional comparator:

```bash
uv run scripts/alignment.py \
  --models Snowflake/snowflake-arctic-embed-m-v2.0 \
  --convert-missing \
  --tei-start \
  --benchmark
```

If TEI is already running:

```bash
uv run scripts/alignment.py \
  --models Snowflake/snowflake-arctic-embed-m-v2.0 \
  --tei-url http://127.0.0.1:8080
```

## Correctness Cases

The benchmark standard is documented in
[`benchmarks/STANDARD.md`](../benchmarks/STANDARD.md). The script includes cases
intended to catch both numerical drift and batching bugs:

- Single English and Chinese inputs
- Mixed Chinese/English batch
- Duplicate inputs in the same batch
- Large length skew inside one batch
- `snowflake_batch_regression`: `["A cute cat....", "A cute cat."]`
- Internal `embeddings.cpp` batch-vs-single comparison for every case

Cosine thresholds should be read as drift tolerances. Invalid output is a
failure; lower cosine is a reported quality metric unless the caller explicitly
uses a CI gate such as `--fail-on-threshold` in scripts that support it.

## TEI

TEI is enabled with either:

- `--tei-url http://host:port` for an existing TEI server
- `--tei-start` to start Docker automatically for models marked as supported in `MODEL_SPECS`

The default Docker image is:

```text
ghcr.io/huggingface/text-embeddings-inference:cpu-1.9
```

TEI support is intentionally explicit. As of this script, `Snowflake/snowflake-arctic-embed-m-v2.0` is marked supported because TEI documents it as an Alibaba GTE model.

## Outputs

Each run writes:

- `scripts/output/alignment_YYYYMMDD_HHMMSS.json`
- `scripts/output/alignment_YYYYMMDD_HHMMSS.csv`
- `scripts/output/alignment_YYYYMMDD_HHMMSS.md`

The Markdown report contains correctness and performance tables. Performance rows include mean latency, p95 latency, throughput, and RSS memory. For auto-started TEI containers, memory is read from `docker stats`.

## Useful Options

```bash
uv run scripts/alignment.py --help
```

Common options:

- `--models ...`: test a subset of model IDs
- `--quantizations fp16 q8_0 q6_k`: test specific GGUF files when present
- `--skip-python`: run C++ internal checks without Python reference
- `--benchmark-batches 1 2 4 8 16`: choose benchmark batch sizes
- `--torch-threads N`: pin Python CPU thread count
- `--cpp-threads N`: pin embeddings.cpp CPU thread count. If omitted, C++ uses `min(8, hardware_concurrency)` and can also be controlled with `EMBEDDINGS_CPP_THREADS`.
